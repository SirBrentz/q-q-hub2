QUIDGET SITE — UPDATE PACK
===========================

Drop the folders into the root of your Quidget site and let them merge.
This pack replaces only the files that changed.

LATEST CHANGES (SEO / Semrush audit fixes)
------------------------------------------
1. Three new files at the site ROOT
   - /robots.txt   — standard crawl directives + sitemap pointer
   - /sitemap.xml  — all 6 public pages listed with lastmod dates
   - /llms.txt     — site summary for AI crawlers (llmstxt.org spec)
   All three were returning HTTP 500 because they didn't exist at the
   server root. Creating them resolves:
     • 2 x ERROR: 5xx errors (robots.txt, sitemap.xml)
     • 1 x WARNING: Sitemap.xml not found
     • 2 x NOTICE: Llms.txt / Robots.txt not found

2. /get-started/index.html
   - Removed a <link> to /assets/css/styles.css — the file doesn't
     exist and the page uses inline styles anyway. Resolves:
       • 1 x ERROR: Broken internal JavaScript and CSS files
   - Added a static <h1> inside the React root that renders before
     hydration. Crawlers see the h1 in the HTML source; React
     replaces it with the wizard UI on page load. Resolves:
       • 1 x WARNING: Missing h1

3. /index.html and /weight-loss/index.html
   - Broke up the static email "care@beautifultelehealth.com" in the
     demo UI mockup with an empty <span></span> between username and
     @ sign. This prevents Cloudflare's Email Obfuscation from
     wrapping the address in /cdn-cgi/l/email-protection URLs that
     Semrush's crawler reports as broken. The displayed text is
     unchanged. Resolves:
       • 2 x ERROR: Broken internal links (1 per page)
